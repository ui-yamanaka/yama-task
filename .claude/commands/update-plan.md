# 企画骨子アップデート

## 使い方

以下の3つを渡してください：
1. **企画骨子ファイル** — 更新対象のMarkdownファイルパス
2. **最新の議論内容** — 議事録・Slackログ・メモなどをそのままペースト
3. **追加資料**（任意） — 参考にすべきドキュメントがあれば添付

例：
```
/update-plan

【企画骨子】docs/2026-05-CSカルテDB企画.md

【議論内容】
（議事録やSlackログをここにペースト）

【追加資料】
（任意）
```

---

## 実行内容

渡された情報をもとに以下を順番に実行してください。

### Step 1: 現状把握
- 企画骨子ファイルを読み込む
- 議論内容・追加資料を読み込む

### Step 2: 差分を分析
議論内容を精査して、企画骨子に反映すべき変更点を特定する：
- 新たに決まったこと
- 変更・修正されたこと
- 削除・除外されたこと
- 未解決のまま持ち越されたこと

### Step 3: 企画骨子を更新
- 変更点を企画骨子に反映して上書き保存
- ファイル末尾に更新履歴を追記：
  ```
  ## 更新履歴
  - YYYY-MM-DD：（変更サマリーを一行で）
  ```

### Step 4: 変更サマリーを出力
以下のフォーマットでチャットに出力する：

```
## 変更サマリー

### ✅ 反映した変更
- （箇条書き）

### ⚠️ 要確認・持ち越し
- （未解決事項があれば）

### 📄 更新ファイル
- ファイルパス
```

### Step 5: GitHubにpush
```bash
git add docs/
git commit -m "docs: {ファイル名} 企画骨子を更新 YYYY-MM-DD"
git push origin main
```

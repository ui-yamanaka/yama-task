# Slackログ保存

## 使い方

チャンネル名を指定して実行してください。

```
/save-slack-log

【チャンネル】#チャンネル名
【期間】（任意：例 "直近3日" "先週" デフォルトは直近7日）
```

---

## 実行内容

### Step 1: Slackからログを取得

Slack MCPツールを使って、指定されたチャンネルの直近のメッセージを取得する。
- 期間が指定されていればその期間、未指定なら直近7日分を対象とする
- スレッド返信も含めて取得する

### Step 2: 内容を分析

取得したメッセージを読み込み、以下を抽出する：

- **決定事項**：「〜にします」「〜で進める」「〜に決まった」など、確定した事柄
- **議論の論点**：議論になったテーマ・対立軸・検討された選択肢
- **ネクストアクション**：「〜をやる」「〜を確認する」「〜までに」などタスクや宿題になった事柄

### Step 3: ファイルに保存

以下のフォーマットで `docs/slack-logs/YYYY-MM-DD-{チャンネル名}.md` に保存する。
ファイル名の日付は実行日、チャンネル名は#を除いた名前を使う。

```markdown
# YYYY-MM-DD #{チャンネル名} ログまとめ

## チャンネル
`#{チャンネル名}`

## 期間
YYYY-MM-DD 〜 YYYY-MM-DD

## 概要
（議論全体を1〜2文で要約）

## 決定事項
- （箇条書き）

## 議論の論点
- **論点名**：（内容・対立軸・結論があれば）

## ネクストアクション
| アクション | 担当 | 期日 |
|-----------|------|------|
| | | |
```

### Step 4: GitHubにpush

```bash
git add docs/slack-logs/
git commit -m "docs: #{チャンネル名} Slackログまとめ YYYY-MM-DD"
git push origin main
```

### Step 5: 完了サマリーを出力

```
## 保存完了

### 📄 保存ファイル
- docs/slack-logs/YYYY-MM-DD-{チャンネル名}.md

### ✅ 決定事項（件数）
### 💬 論点（件数）
### 📌 ネクストアクション（件数）
```
